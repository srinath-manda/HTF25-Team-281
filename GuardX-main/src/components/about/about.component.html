About GuardX

An Intelligent, Adaptive Web Application Firewall for the Next Generation of Threats.

Project Purpose

This project was developed for the COSC - Hacktoberfest : Hackathon. Our goal was to move beyond the limitations of traditional, signature-based firewalls by creating a smart, proactive, and self-improving security system. GuardX is a high-fidelity simulation that demonstrates how modern AI can be leveraged to create a Web Application Firewall (WAF) that not only blocks known threats but also learns from new, unseen attacks to dynamically harden its defenses in real-time.

The Adaptive Defense Engine

The core of GuardX is its AI-powered Adaptive Defense Engine. Unlike traditional WAFs that rely on manually updated lists of attack patterns, GuardX uses Google's Gemini API to analyze and respond to threats intelligently.

  1. Threat Identification: The WAF inspects all incoming traffic. When it blocks a request that matches a known threat pattern (e.g., a basic SQL Injection), it logs it and prevents it from reaching the target application.
  2. Analysis of the Unknown: When a request is blocked that does not match a pre-defined signature, the WAF flags it as a potential zero-day or obfuscated attack. The malicious payload is securely sent to the Gemini API for deep analysis.
  3. AI-Powered Rule Generation: The Gemini model analyzes the payload's structure, intent, and attack vector. It then generates a concise, human-readable description for a new, dynamic security rule designed to mitigate this specific threat vector.
  4. Real-Time Deployment: This newly generated rule is instantly deployed to the WAF's rule set, which you can see on the "Adaptive Defense" page. The firewall is now equipped to block this new attack pattern and its variants, effectively learning and adapting from the threats it encounters.
