Web Application Threat Model

Understanding the threats GuardX is designed to mitigate. This model outlines common attack vectors against modern web applications.

Cross-Site Scripting (XSS)

XSS attacks occur when malicious scripts are injected into trusted websites. The attacker's script is then executed in the victim's browser, allowing the attacker to hijack user sessions, deface web sites, or redirect the user to malicious sites.

Potential Impact

  • Session hijacking and cookie theft
  • Website defacement
  • Keystroke logging
  • Redirecting users to malicious sites

GuardX Mitigation

  • Inspects payloads for <script> tags.
  • Blocks common event handlers like onerror.
  • AI analysis detects obfuscated script patterns.

SQL Injection (SQLi)

SQL Injection is a code injection technique used to attack data-driven applications. Malicious SQL statements are inserted into an entry field for execution, allowing an attacker to bypass authentication, view, modify, or delete data in a database.

Potential Impact

  • Authentication bypass
  • Complete data exfiltration
  • Data corruption or deletion
  • Full database server takeover

GuardX Mitigation

  • Blocks classic SQLi syntax like ' OR 1=1.
  • Detects SQL comment characters --.
  • Identifies keywords such as UNION, SELECT.

Path Traversal

Also known as "dot dot slash," this attack aims to access files and directories stored outside the web root folder. By manipulating variables that reference files with ../ sequences, an attacker can navigate up the directory tree and access sensitive system files.

Potential Impact

  • Exposure of sensitive system files (e.g., /etc/passwd)
  • Disclosure of application source code
  • Access to configuration files and credentials

GuardX Mitigation

  • Scans request paths for directory traversal patterns.
  • Normalizes paths to resolve sequences before checking.
  • Blocks requests containing suspicious file access attempts.

Brute-Force Attacks

A brute-force attack is a trial-and-error method used to obtain information such as a user password. In this type of attack, automated software is used to generate a large number of consecutive guesses as to the value of the desired data.

Potential Impact

  • User account compromise and takeover
  • Denial of Service by overwhelming the login system
  • Credential stuffing success if passwords are reused

GuardX Mitigation

  • Implements rate limiting on login endpoints.
  • Temporarily blocks IPs after too many failed attempts.
  • Adaptive AI can identify automated attack patterns.
